**일시**: 2026-03-04 **단계**: Phase 3 - 봇 탐지 미들웨어 구현 + 전체 코드 리뷰 수정

## 1. 봇 탐지 미들웨어 구현

### 신규 파일

| 파일 | 역할 |
| --- | --- |
| `utils/bot-detect.ts` | User-Agent 패턴 매칭으로 AI/차단/일반 봇 분류 |
| `utils/supabase-admin.ts` | Service-role Supabase 클라이언트 + `logBotAccess()` |
| `routes/_middleware.ts` | Fresh 미들웨어: 차단→403, AI봇→비동기 로깅 |

### 봇 분류 규칙

| 분류 | 봇 | 동작 |
| --- | --- | --- |
| AI 봇 | GPTBot, ChatGPT-User, OAI-SearchBot, ClaudeBot, Claude-Web, PerplexityBot, Google-Extended, Googlebot, Bingbot | 동일 HTML 응답 + bot_logs 비동기 기록 |
| 차단 봇 | Bytespider, CCBot | 403 Forbidden + bot_logs await 기록 |
| 일반 | 그 외 | 동일 응답, 로깅 없음 |

### 설계 원칙

- **클로킹 방지**: AI 봇과 일반 유저에게 동일한 HTML 응답
- **비동기 로깅**: AI 봇은 fire-and-forget + `.catch()` 에러 핸들링
- **차단 봇 await**: Deno Deploy에서 응답 후 비동기 작업 유실 방지
- **Graceful fallback**: `SUPABASE_SERVICE_ROLE_KEY` 미설정 시 로깅 스킵, 앱 정상 동작

## 2. CodeRabbit 리뷰 수정 (Phase 1-3 전체)

Phase 1-2에서 CodeRabbit 리뷰 없이 작성된 코드 + Phase 3 코드에 대해
`coderabbit review --plain -t uncommitted`로 97건 리뷰 수행 후 수정.

### Critical/High 수정

#### 1. 차단봇 logBotAccess() await 누락
- `routes/_middleware.ts`: 차단봇 403 응답 전 `logBotAccess()`에 `await` 추가
- Deno Deploy에서 응답 전송 후 비동기 작업이 유실될 수 있는 문제

#### 2. AI봇 fire-and-forget .catch() 누락
- `routes/_middleware.ts`: AI봇 로깅에 `.catch()` 핸들러 추가
- 에러가 조용히 무시되는 문제 방지

#### 3. RLS 주석 불일치
- `scripts/schema.sql`: "RLS 비활성화" 주석 → "RLS 활성화"로 수정
- 실제 코드는 `ENABLE ROW LEVEL SECURITY`인데 주석이 반대

#### 4. bot_logs/click_logs SELECT 정책 누락
- `scripts/schema.sql`: service_role용 SELECT 정책 추가
- 로그 조회 시 RLS에 의해 차단되는 문제

### Medium 수정

#### 5. FK/로그 인덱스 추가
- `scripts/schema.sql`: `products.brand_id`, `products.is_active` 인덱스
- `product_ingredients`, `ingredient_symptoms` FK 인덱스
- `bot_logs.timestamp`, `click_logs.timestamp/product_slug` 인덱스

#### 6. OG 메타 태그 누락
- `routes/_app.tsx`: 기본 title, description, og:site_name, og:type 추가
- `routes/products/[slug].tsx`: og:title, og:description, og:type 추가

#### 7. 별점 클램핑
- `routes/products/[slug].tsx`: `Math.min(5, Math.max(0, Math.round(product.rating)))`
- 잘못된 rating 데이터에 의한 별 6개 이상 렌더링 방지

#### 8. JSON-LD 상대 URL → 절대 URL
- `routes/symptoms/[slug].tsx`: `BASE_URL` 상수 추가, JSON-LD product URL 절대경로화

#### 9. 브랜드 fallback 한국어화
- `routes/products/index.tsx`: "Unknown Brand" → "알 수 없는 브랜드"

#### 10. API 에러 응답 status 필드
- `routes/api/products/[slug].ts`: `{ error: "..." }` → `{ error: "...", status: 404 }`

#### 11. Supabase 타입 단언 수정
- 3개 파일: `data as Type` → `data as unknown as Type`
- Supabase 조인이 배열을 반환하는데 인터페이스는 단일 객체로 정의한 불일치 해소

#### 12. react-no-danger 린트 규칙 제외
- `deno.json`: lint rules exclude에 `react-no-danger` 추가
- JSON-LD `<script>` 태그에 `dangerouslySetInnerHTML`은 필수적 사용

### 의도적 스킵 (Phase 4-5에서 처리)

- API 라우트 CORS/Rate Limiting/Auth → Phase 5 배포 시
- scripts/ 타입 어노테이션 강화 → 우선순위 낮음
- 구조화 로깅 (pino 등) → Phase 5
